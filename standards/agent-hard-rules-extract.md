# Agent Hard-Rules Extract for Chinese Math Lecture Notes

这是从私有 agent 硬规则中抽出的公开版，只保留中文数学讲义生成相关规则。

## 中文讲义质量是硬门槛

- 中文数学讲义的中文 prose quality 是硬门槛，不是润色偏好。
- 正式讲义必须以紧凑的 `## 一页摘要` 开头，摘要要先说明中心问题、主答案、阅读路径和边界。
- 正式讲义必须在可点击 `## 目录` 后、第一处实质章节前放前置路线图。
- Mermaid 路线图是有效且推荐的路线图；只要清楚，不应无故替换成表格。
- 使用英文术语、论文原名、API 名或非标准标签时，必须有 `术语约定` / `符号与术语`。
- 普通解释性英文应翻译；只保留精确 API 名、包名、类名、论文标题、定理标签、代码标识符、标准缩写或原文引用。
- 不得引入私有 agent 术语、英文拼接标签或工作流黑话，除非立即定义、说明动机，并给出正例/非例子。

## 学习负荷与复现测试

- validator 和 proof audit 必须检查学习负荷 / 重点密度。
- 正式讲义应尽早进入命名的 `贯穿例子` / `核心例子`，并在定义、证明、边界和复现测试中持续复用。
- 每份正式讲义必须包含具体 `复现测试`：输入、预期输出和通过标准。
- 必须覆盖 `主题主干`：用户点名概念、源材料中心定义/证明工具、题目承诺的 workflow 必须进入实质章节；若不覆盖，必须在一页摘要里明确 scope out。

## 前置结构不能膨胀

- 一页摘要、目录、前置路线图和术语约定只负责定向；之后应快速进入核心计算或证明。
- 不得用长路线表、meta checklist 或定义堆替代数学主体。

## μP / Tensor Programs 特别规则

如果讲义声称教学 μP 设置或训练动态，必须覆盖：

- `abc 参数化`
- μP 初始化和学习率缩放
- 相关时的高斯条件化 / 初始输出条件化
- 固定步数的多步递推

只计算初始化核不等于完成 μP 训练动态讲义。
